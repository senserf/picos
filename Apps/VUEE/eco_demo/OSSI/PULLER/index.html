EcoNetwork EMS Data Extraction

Usage:

The From field should include a date/time in one of several popular formats, e.g., Apr 29, 2009 12:33, 17:15:03 (refers to the current day), 09-04-29 14:44, and so on. This field is mandatory.

The Upto field may include a second (full) date/time string (specifying the upper limit) or a number of days (counted from the From date/time). If the field is empty, it translates into 1 day.

The Rig Id field is mandatory and must include the EMS device number (415, 463, and so on).

The Collector field may be empty (all collectors) or may include a single collector number. If you put the word events into it, you will see event messages (i.e., all received messages that are not samples).

Notes:

The primary time stamp of a sample/event (used for searching) is the Delivery Time, as marked by the EMS database upon the reception of the sample from the satellite. This is the only time that is always and unconditionally reliable. Note that it is GMT at present. We can easily offset this time stamp as well as the time stamps of the collectors (the configuration setup provides a parameter for that).

The TS field in an extracted CSV file tells whether the collector's time stamp is reliable or not. The letter Y means that the collector has acquired its time from the gateway, and N that it has not.

The database is updated every 60 seconds by pulling any fresh batch of data from the EMS site.
